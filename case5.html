Blurt-Ed!

Blurt-Ed! – Prototyping an Iterative Drafting Interface

Abstract

This project is currently focused on the development of an interface that can support respeaking as an alternative to precise editing, as an alternative way to reduce the hands-and-eyes demand on current speech-to-text systems. This project aims to explore a Blurt-Review-Respeak (BRR) Model, which allows users to produce large amounts of text and edit the text via speech on a clause-level granularity while utilizing their original sentence as support for their newer composition. By supporting the selection, respeaking and deletion of text via screen-based gesture commands like tap, drag and slide, we aim to understand whether respeaking can be used as a viable alternative to keyboard-based editing on speech-to-text systems as well as how to reduce the user’s reliance on the keyboard and other peripheral devices for editing in order to provide a more hands-free experience.

Study Outcome

Along with the Did I Say That? study, this exploration of an interface that supports a new way of producing oral compositions is a part of our initial set of studies that work towards developing a more intuitive speech-to-text system based on how humans cognitively produce and process speech, which in turn was inspired by the lack of existing literature or exploration of speech in the field of HCI. This project is still a work-in-progress, so stay tuned for updates!!
